# EEE 404/591 — Real Time DSP

**Instructor:** [[chao-wang]] (Dr. Chao Wang, ASU)
**Lab platform:** STM32F407G-DISC1 kit
**Software:** STM32CubeIDE, MATLAB

## Overview
Digital Signal Processing with a real-time / embedded twist. The first big arc is the **Fast Fourier Transform** — how to compute a DFT fast enough to run on a microcontroller, then how to interpret and window the results. Both floating-point and fixed-point implementations are emphasized because the STM32 can do both and fixed-point is faster.

## Roadmap (FFT module — what we've covered)

1. **[[dft]]** — what the Discrete Fourier Transform actually is ($N^2$ complexity)
2. **[[fft]]** — same transform, computed fast via decomposition
3. **[[twiddle-factor]]** — the complex roots of unity that stitch it all together
4. **[[decimation-in-time]]** — the DIT algorithm, even/odd split
5. **[[butterfly]]** — the atomic FFT building block
6. **[[bit-reversed-order]]** — why DIT inputs need reshuffling
7. **[[fft-scaling]]** — how to prevent overflow
8. **[[dft-bin-interpretation]]** — which bin = which frequency
9. **[[frequency-resolution]]** — $\Delta f = f_s/N$
10. **[[conjugate-symmetry]]** — why real-valued DFTs are half-redundant
11. **[[idft]]** — computing inverse DFT using forward FFT + two conjugations
12. **[[real-valued-fft]]** — packing a real signal into a half-size complex FFT
13. **[[fixed-point-arithmetic]]** — shift-factor Q15 scaling for embedded
14. **[[stft]]** — Short Time Fourier Transform (analyzing events in time)
15. **[[window-function]]** — multiplying by a window before FFT
16. **[[rectangular-window]]**, **[[hamming-window]]**, **[[hann-window]]**, **[[bartlett-window]]**
17. **[[spectral-leakage]]** — the reason windowing exists
18. **[[dtft]]** — the continuous-frequency parent of the DFT (HW5)
19. **[[window-resolution-criterion]]** — when can a window separate two tones? (HW5)
