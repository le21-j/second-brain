# Sionna RT 2023 — Differentiable Ray Tracing for Radio Propagation Modeling

**Authors:** Jakob Hoydis, Fayçal Aït Aoudia, Sebastian Cammerer, Merlin Nimier-David, Nikolaus Binder, Guillermo Marcus, Alexander Keller (NVIDIA Research). **arxiv:2303.11103** (March 2023). Mirrored at `raw/articles/ml-phy/pdfs/sionna-rt-2023.pdf`.

## TL;DR
**Sionna RT** brings full **differentiable ray tracing** to the wireless world: every electromagnetic ray bounce in a 3D scene becomes a tensor op whose gradient is well-defined with respect to scene geometry, material parameters, transmitter/receiver placement, and antenna patterns. Built on Mitsuba 3 + Dr.Jit; framework-agnostic across TensorFlow / PyTorch / JAX / NumPy. Unlocks the **wireless digital twin** vision — calibrate a simulated scene against real measurements by gradient descent.

## Key contributions

1. **Differentiable specular + diffuse reflection + diffraction + refraction** in a unified physics-grounded ray-tracing framework. Earlier wireless RT (Wireless InSite, Remcom) was non-differentiable; classical channel models (3GPP 38.901) are statistical. Sionna RT bridges both.
2. **Path-tree representation.** For each TX–RX pair the tracer returns the full set of paths (time of arrival, angle of departure / arrival, complex amplitude, polarization), batched and differentiable.
3. **Coverage maps / Radio Maps.** Heatmaps of path loss, RSS, SINR over a scene, generated by exhaustive RX-grid ray-tracing. Foundation for site-specific ML downstream.
4. **Calibration recipe.** Sionna RT + a measured channel-impulse-response dataset → gradient descent over **material conductivity, permittivity, surface roughness** → simulated scene matches measurements. Productionized in [[paper-diff-rt-calibration-2024]] (Hoydis IEEE TMLCN 2024 — separate paper, not yet ingested).

## Methods

- **Mitsuba 3** (Wenzel Jakob's Monte Carlo light transport library) + **Dr.Jit** (just-in-time differentiable tracing) provide the auto-diff backbone.
- Scenes loaded as **Mitsuba XML** or **glTF/GLB**; OSM building footprints are the most common source.
- Ray tracer supports **specular reflection, refraction, diffuse scattering** (Phase 1 release, March 2023). **Diffraction added in v1.2 (late 2023)**. **RIS support added v0.18 → v1.x**.

## Results

- **3GPP UMi calibration.** Demonstrates that an OSM-loaded urban scene can be calibrated to within ~1 dB RMS path-loss error of measured drive tests by optimizing a handful of material parameters — the first time this had been done with gradient descent rather than manual tuning.
- **Site-specific neural receiver (Phase 4 M10 of the roadmap).** Pretraining an NRX on 3GPP UMi and fine-tuning on a Sionna RT custom scene gives 1–2 dB BLER gain at the target site.

## Portfolio move (Phase 4 M10)

> Site-specific NRX in Sionna RT — custom OSM scene; pretrain on 3GPP UMi; fine-tune per-BS; show site-specific BLER gain.

Steps:
1. Pick an OSM area (e.g., a few-block downtown area near ASU campus).
2. Load via Sionna RT, place a BS + grid of UE positions.
3. Compute coverage map; generate channel realizations.
4. Pretrain NRX on 3GPP UMi (cross-environment baseline).
5. Fine-tune NRX on the Sionna RT scene; evaluate on held-out RX positions.
6. Report site-specific BLER vs. cross-environment baseline.

## Questions raised
- **Diffuse scattering parameter inference** — Sionna RT exposes the parameters but inferring them from limited measurements is an open problem (one of Hoydis's later 2024 papers).
- **Computational cost.** Coverage maps over a city block at 28 GHz need 1–2 GPU-hours per scene. Use AODT / digital twin acceleration for larger scenes.
