# Neural decoder

## In one line
**A neural decoder is a learned replacement for (or learned augmentation of) a classical channel decoder — typically [[belief-propagation]] for [[ldpc-codes]] or polar codes — where the message-passing weights are trained end-to-end on bit-error-rate.**

## Example first

**Neural BP for a (7,4) Hamming code (Nachmani et al. 2018).** Take the parity-check graph (4 variable nodes + 3 check nodes). Run sum-product BP for 5 iterations. Now **add a learnable scalar weight to every message edge** — both variable-to-check and check-to-variable. Train these weights to minimize BCE between the decoder output and ground-truth bits over a training set of received noisy codewords.

Result: the learned BP achieves **0.2–0.5 dB BER gain** over classical BP at the same iteration count — the message weights have learned to cancel cycle-induced errors in the loopy graph that classical BP can't handle. **The learned decoder runs at the same complexity as classical BP.**

This is the canonical demonstration that **decoders are differentiable computational graphs** and can be trained.

## The idea

Channel decoders all share the same skeleton: **iterate a message-passing rule over a graph until convergence**. Classical decoders pick the messaging rule by hand (sum-product / min-sum / Viterbi). **Neural decoders learn the rule.**

Three flavors:
1. **Weighted BP / parameterized BP** (Nachmani 2018) — keep the BP graph structure; multiply each edge message by a learnable scalar; train over BER.
2. **Pruned + quantized neural BP** (Buchberger 2020) — additionally learn which edges to drop and how to discretize; matches classical BP performance with 10× less compute.
3. **Graph-neural-network decoder** ([[graph-neural-network]]) — fully replace BP with a GNN whose message-passing layer is general (not BP-specific). Generalizes beyond BP.

## Formal definition

Classical sum-product BP message:

$$\mu_{f \to v}(v) = \sum_{\mathbf{x}_f \setminus v} f(\mathbf{x}_f) \prod_{u \in N(f) \setminus v} \mu_{u \to f}(u)$$

Neural BP injects learnable weights $w_{f,v}$:

$$\tilde{\mu}_{f \to v}(v) = w_{f,v} \cdot \mu_{f \to v}(v)$$

(in LLR domain — multiplication ↔ scaling). Train end-to-end with BER or BCE loss over a synthetic training set generated by the channel + code combination.

For LDPC: codeword length 1024 → 1024-node parity-check graph → ~3000 learnable weights → trainable in <1 GPU-hour.

## Common mistakes

- **Training without enough SNR diversity.** A neural decoder trained at SNR=2 dB will overfit to that operating point. Train across SNR sweeps.
- **Not normalizing message magnitudes.** Without LayerNorm or message-magnitude clipping, deep BP iterations blow up. Buchberger 2020 makes this explicit.
- **Over-claiming generalization.** Classical BP is **code-agnostic** (works for any parity-check matrix). Most neural decoders are **code-specific** (re-trained per code). The minority of GNN-based decoders generalize across codes — explicitly note which kind in any reproduction.
- **Forgetting the regularity advantage.** 5G LDPC has a structured (regular) graph; cycle-induced errors are bounded; neural BP gives <0.5 dB gain. On unstructured / loopy graphs it can give 1–2 dB. Don't extrapolate small-code wins to big-code wins.
