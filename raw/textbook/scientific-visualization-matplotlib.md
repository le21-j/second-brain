# Scientific Visualization: Python + Matplotlib — Nicolas Rougier

**Category:** Scientific Python (visualization)
**Status:** FREE (open access book) — not yet mirrored
**URL:** https://github.com/rougier/scientific-visualization-book
**License:** CC BY-NC-SA 4.0 (for the text); code is BSD 2-clause
**Roadmap phase:** ongoing — you need publication-quality figures for papers

## Topic coverage
- Anatomy of a matplotlib figure (Figure/Axes/Artist hierarchy)
- Layout, typography, color (perceptually uniform colormaps, colorblind-safe palettes)
- Common traps and their fixes
- Gallery-grade case studies: 3D, animations, dashboards, network visualizations
- Code patterns that produce figures good enough for Nature/Science-style print

## Why it's on the roadmap
Papers and GitHub READMEs live or die on their headline figure. Rougier's book is the open-access gold standard for making matplotlib produce publication-quality work. Contrasts with default matplotlib, whose out-of-the-box output is ugly.

## The move for Jayden's portfolio
Every Phase 2+ repo README should have a **headline figure in the first screen**. That figure should be made with this book's patterns: clear typography, a named colormap you chose (not the default), labeled axes, zero-waste whitespace, saved as SVG or 300-DPI PNG.
